Noções de design para desenvolvedores: processos e dicas CSS para melhorar seu web design

Tradução não tão literal deste artigo.

Tecnicamente qualquer pessoa pode cozinhar. Mas há uma grande diferença entre saber como preparar uma boa refeição e jogar os ingredientes na panela esperando que saia algo bom dali. Assim como no desenvolvimento web, você pode até conhecer os ingredientes —span, background-color, .heading-1— mas não são todos que conseguem transformar esses ingredientes num website lindo e fácil de usar.

A partir do momento que se usa HTML e CSS, você está criando (dando forma e estrutura) o conteúdo que será apresentado para que ele possa ser entendido por outra pessoa. As pessoas vêm projetando há séculos. Nesse tempo, foram desenvolvidos princípios que se aplicam às interfaces digitais de atualmente. Esses princípios se destacam em 3 áreas principais:

  • #Tipografia: como as palavras são exibidas;
  • #Espaçamento: como o conteúdo é organizado;
  • #Cor: como a personalidade é adicionada;

Vamos descobrir como cada um desses ingredientes pode ser usado, com propriedades e diretrizes CSS, para eliminiar o "jogo de adivinhação" da Web Design.

Tipografia

Sites de fácil leitura não acontecem por engano. Na verdade, Taimur Abdaal escreveu um artigo inteiro sobre o tópico, repleto de conselhos para desenvolvedores que trabalham com tipografia. Vamos nos concentrar especificamente em dois princípios fundamentais de design que podem ajudá-lo na hora de exibir palavras para que visualmente fique mais agradável e fácil de ler: repetição e hierarquia.
